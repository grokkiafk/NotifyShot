## 🎮 Что это такое

Записал длинный геймплей (например, смену медиком на GTA RP) и нужно **достать скриншоты конкретного момента** — каждый раз, когда всплывало уведомление «Вы реанимировали», «Вы вакцинировали» и т.п.?

Раньше это значило часами перематывать запись вручную. **NotifyShot делает это за тебя:**

> Кидаешь видео → выбираешь, что искать → получаешь папку с готовыми скриншотами, по одному на каждое срабатывание.

Идеально для отчётов, статистики, монтажа, портфолио.

## ⬇️ Как пользоваться (3 шага)

1. **[Скачай архив](https://github.com/grokkiafk/NotifyShot/releases/latest)** и распакуй куда угодно.
2. Запусти **`NotifyShot.exe`** — ставить ничего не надо, всё уже внутри.
3. В окне:
   - **Видео** → добавь свои записи (или целую папку);
   - **Уведомление** → выбери, что ловить;
   - жми **СТАРТ** 🚀

Готовые скриншоты появятся в папке `Рабочий стол\NotifyShot` (можно поменять). По окончании программа сама предложит её открыть.

> ⚠️ Не вытаскивай `NotifyShot.exe` из папки — рядом должна лежать папка `bin`.

А можно **добавить свой** — для любого уведомления с повторяющимся текстом (награды, штрафы, аресты, киллы, что угодно).

## ➕ Свой профиль (за минуту)

В разделе **«Профили»** → **«Создать новый»**:

1. **Загрузи скриншот** игры с уведомлением (или возьми кадр прямо из видео).
2. **Обведи рамкой** иконку и неизменный текст уведомления — **без** меняющихся ников и цифр (например, обведи «получили награду», а сумму не трогай).
3. Дай название и сохрани. Профиль сразу появится в списке.

## ⚙️ Настройки (если что)

- **Чувствительность** — Низкая / Средняя / Высокая. Ловит лишнее → подними; что-то пропускает → понизь.
- **Кадров на событие** — сколько скринов сохранять вокруг момента (1–5), чтобы выбрать лучший ракурс.

По умолчанию стоят настройки, на которых всё работает «из коробки» — менять не обязательно.

## ❓ Частые вопросы

<details>
<summary><b>Это безопасно? Не прилетит бан?</b></summary>

NotifyShot **никак не лезет в игру** — он просто читает твои видеофайлы на компьютере и режет из них картинки. С игрой, лаунчером и серверами он вообще не взаимодействует.
</details>

<details>
<summary><b>Сколько стоит?</b></summary>

Бесплатно и с открытым исходным кодом (лицензия MIT).
</details>

<details>
<summary><b>Нужен интернет?</b></summary>

Нет. Всё работает локально, ffmpeg уже встроен.
</details>

<details>
<summary><b>Какие видео понимает?</b></summary>

MP4, MKV, MOV, AVI и другие. Любое разрешение записи (720p / 1080p / 1440p…).
</details>

<details>
<summary><b>Долго обрабатывает?</b></summary>

Быстро: программа смотрит не все кадры, а только опорные (≈1 раз в 2 секунды), поэтому час записи разбирается за пару минут.
</details>

# 🛠️ Техническая часть

<sub>Дальше — для тех, кто хочет собрать из исходников, понять, как оно устроено, или сделать профили руками.</sub>

## Как это работает под капотом

1. **Декод только ключевых кадров.** Через `ffmpeg -skip_frame nokey` декодируются лишь keyframe'ы (в типичной записи — раз в ~2 c). Это на порядок меньше работы, чем перебирать все 60 fps, а уведомление висит несколько секунд и гарантированно попадает на keyframe.
2. **Шаблонное сопоставление.** Каждый ключевой кадр кропается до зоны, где появляется плашка, и сравнивается с шаблоном (картинкой-образцом) через `cv2.matchTemplate` (`TM_CCOEFF_NORMED`). Шаблон масштабируется под разрешение видео, плюс небольшой перебор масштабов (jitter) на случай нестыковок.
3. **Кластеризация и вырезка.** Подряд идущие кадры выше порога объединяются в одно «событие», и из исходного видео в полном разрешении вырезается лучший кадр (`ffmpeg -ss … -frames:v 1`) в PNG.

Прогресс честный: анимированная полоса на этапе анализа, реальные проценты на распаковке и поиске. Временные файлы (кропы кадров) удаляются автоматически через `finally` — ничего не копится.

## Формат профиля

Профиль = JSON + картинка-шаблон рядом (`templates/*.json` + `*.png`):

```json
{
  "name": "Вакцинация — Majestic",
  "image": "vaccine_majestic.png",
  "box": [1018, 1323, 194, 58],
  "src_size": [2559, 1430],
  "pad": [2.0, 5.0, 1.5],
  "threshold": 0.90
}
```

- `box` — `[x, y, w, h]` рамки уведомления на исходном скриншоте;
- `src_size` — размер того скриншота (чтобы пересчитать координаты под любое видео);
- `pad` — насколько расширить зону поиска (по ширине, вверх, вниз), в долях от рамки;
- `threshold` — порог совпадения (0–1); ползунок чувствительности лишь сдвигает его.

Встроенные профили лежат внутри `.exe`; свои (созданные мастером) — в папке `templates` рядом с программой. Чтобы поделиться профилем — просто скинь `.json` + `.png`.

## Сборка из исходников

```bat
pip install -r requirements.txt pyinstaller
python get_ffmpeg.py        :: положит ffmpeg в bin\ (или возьми сборку BtbN — быстрее)
build.bat                   :: соберёт dist\NotifyShot.exe
```

Запуск без сборки: `python app.py` (нужен ffmpeg в `PATH` либо в `bin\`).
Иконку можно перегенерировать: `python make_icon.py`.

## CLI (для тестов и подбора порога)

```bat
python detector.py "video.mp4" --profile templates\pmp_majestic.json --out out
python detector.py "video.mp4" --profile templates\pmp_majestic.json --scan   :: топ совпадений + статистика
```

Полезные флаги: `--sensitivity low|med|high`, `--threshold 0.9`, `--frames-per-event 3`.

## Структура проекта

```
app.py            GUI (CustomTkinter, безрамочное окно в стиле лаунчера Majestic)
detector.py       ядро: ffmpeg + OpenCV, поиск и вырезка (импортируется и работает из CLI)
make_icon.py      генерация иконки
templates/        профили уведомлений (json + png)
get_ffmpeg.py     загрузка ffmpeg в bin/
build.bat         сборка в один .exe (PyInstaller)
```

## Требования

- **Релиз:** Windows 10/11, больше ничего не нужно (ffmpeg внутри).
- **Из исходников:** Python 3.10+, пакеты `opencv-python`, `numpy`, `Pillow`, `customtkinter` (+ `tkinter` из стандартной поставки). Ядро (`detector.py`) кроссплатформенное — на Linux/Mac нужен системный ffmpeg.
